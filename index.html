Pros of using D3:

  • Powerful
  • Flexible
  • Performant
  • Popular and well-established

Cons of using D3:

  • Steeper learning curve
  • More complex
  • Takes longer to make a basic chart
  • Have to give talks about using d3 + React

Pros of using charting libraries:

  • Fast
  • Simpler to learn
  • More readable
Good for prototyping

Cons of using charting libraries:

  • Limited features
  • Restrictive APIs
  • Unnecessary features -> bloat
  • Variable performance
It can take some time to work out whether the API will suit your needs

D3 + React?

What shall we use for the DOM? React is good for DOM rendering, but so is D3. D3 is great for animation and has some great out-of-the-box SVG helpers for tricky things like axes. But using just D3 can make your chart less structured by putting the whole chart in one component, leading to tight-coupling between the individual parts. It's also a bit less readable than React's declarative syntax.
The Hybrid Approach. I prefer the structure and readability that comes by using React components, but I also like using the transitions directly on the DOM elements. So what I do is I combine both approaches: Let React render static components, and use D3 for everything that needs to be animated. Mix and match, see what works for you.
